The Easy Animator

A Fully Editable Animation Creator

⊳ Overview

  • It is a desktop application GUI designed to allow a user to create an edit

    an Animation from scratch

  • It was created with a Partner.

⊳ Implementation Details

  • Easy Animator is developed in Java using the Java Swing GUI widget
    toolkit.
  • It is developed using the MVC (Model View Controller) Architecture.
  • The Adapter Pattern and the effective seperation between
    Model, View, and Controller allows the application to output a different
    GUI depending on the Command Line Arguments.
    Command Line Argument Implementation:
    1. Edit View: The fully implemented GUI
    2. Provider View: Displays a GUI that adapts our Model and Controller
      to another groups implementation of the View component.
      This is done with the Adapter Pattern.
    3. Visual View: Displays only the Animation, without any of the Playback
      features or Controls that allow the Animation to be modified.
    4. SVG View: Provides an SVG file with the implementation of the animation
    5. Text View: Provides a Text File that describes the Animation.

⊳ Features & Applications

  • Create/Display a specified Shape (Circle, Square, Rectangle, Oval)
    and define its properties (Name, Size, Color, Position)
  • Remove any selected Shape by specifying the Shapes name
  • Animate any existing Shape by:
    1. Deleting one or multiple of the Shapes KeyFrames
    2. Adding a KeyFrame for the Shape by specifying Name, Time, Size,
      Position, Color
    3. Modifying an existing KeyFrame by changing its Time, Size,
      Position, or Color.
  • Animation Playback Features:
    1. Starting, pausing/resuming, and restarting an Animation at any
      point in time
    2. Increasing or decreasing the speed of an animation by using
      the Speed-Bar
    3. The ability to scrub the Animation using the Scrubber.

Minesweeper

A Full-Stack Implementation of the Classic game of Minesweeper

⊳ Overview

  • It is a desktop application GUI that implements the classic game of Minesweeper

  • It was created with a Partner.

⊳ Implementation Details

  • Minesweeper is built in Java using the Javalib library.
  • It uses iteration to implement the Flow-effect when the user clicks
    on a Cell in which the adjacent cells do not have a number/bomb.
  • This Flow Effect is used to find the next cell/cells that have an adjacent bomb.

⊳ Features & Applications

  • The ability to fully play a game of Minesweeper with most of its features
  • Features implemented:
    1. Flag any Cell in the board by right-clicking on the selected Cell
    2. Clicking on a cell to display a number or a bomb.
    3. When a Cell does not have a bomb, it displays a number
      indicating the number of adjacent bombs to the selected cell.
    4. If the user clicks on a Cell with a bomb the game ends
      and all of the Cells containing bombs are revealed

Light 'em All

A Fully Editable Animation Creator

⊳ Overview

  • It is a GUI that implements the game of Connect the Wires.

  • This Game consists of GamePieces on a fractal-like board generated
    with a subdivision algorithm which represent different types of Wire shapes.
    When clicked, the gamepiece rotates to change the orientation of the wire.
  • The board also has a GamePiece in which there is a power-station that
    has a finite radius.
  • The power station can be moved with the arrow keys, but only to GamePieces
    in which the wires are correctly oriented so that they connect.
  • The purpose of the Game is to light up all all of the wires by connecting them
    and strategically placing the power station so that the radius reaches
    every GamePiece.
  • Every time the game is played the GamePieces are oriented/rotated randomly
    in order to generate a "random" board.
  • It was created with a Partner.

⊳ Implementation Details

  • Light 'Em All is developed in Java using the Java Swing GUI widget
    toolkit.
  • The fractal-like GamePiece board is generated by using
    a subdivision algorithm
  • The game implements Kruskal's algorithm in order to create
    a "random" board.
  • The light-up behavior, although not completely developed, implements
    the Breadth-first search algorithm in order to find the accurate radius for
    the power station.

⊳ Features & Applications

  • The ability to rotate each GamePiece/Wire by clicking on it in order
    to be able to connect the Wires.
  • Implementation of arrow keys movement in order to move the power station
    through the connected wires.
